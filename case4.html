Automation

Chrome Extension — 반복 입력 8→2단계 자동화

문제
재배송·반품 처리 시 반복 입력과 귀책 기준값 오입력
재배송·반품 처리는 매 건마다 CRM에서 동일한 정보를 반복 입력하는 구조였습니다. 단계가 많아 처리 시간이 길었고, 수동 입력 과정에서 귀책 기준값 오입력 리스크가 있었습니다.
해결
Claude를 활용해 Chrome Extension을 직접 구성하고 실무에 적용
Claude를 활용해 Chrome Extension을 직접 구현했습니다. 반복 입력 단계를 자동화하고, 귀책 기준값을 사전 정의된 기준에 따라 자동 반영해 오입력을 구조적으로 차단했습니다. 처리 단계가 8단계에서 2단계로 축소됐으며, 기준값 오류 없이 일관된 처리가 가능해졌습니다.

AI Usage

생성형 AI — 운영 실무의 파트너로 활용

생성형 AI를 단순 답변 생성이 아니라, 운영 기준을 만들고 문서화하고 검토하는 실무 과정에 직접 활용했습니다.

Claude/GPT는 초안 작성, 문서 구조화, 자동화 로직 설계에 활용했고, NotebookLM은 회사 문서·약관·계약서 원문을 업로드해 조항과 근거를 확인하는 검증 도구로 활용했습니다. 최종 문구와 기준은 실제 운영 흐름과 내부 책임 범위에 맞춰 사람이 수정했습니다.

활용 영역 구체적 활용 방식
FAQ 구조화 반복 VOC 유형별로 FAQ 초안을 구조화하고, 고객 언어와 운영 기준 사이의 표현 간극을 줄이는 문구 검토에 활용
응대 기준 초안 작성 새로운 이슈 유형 발생 시 응대 기준 초안을 빠르게 생성하고, 실제 케이스에 맞게 수정해 SOP에 반영
계약서·약관 분석 NotebookLM에 원문을 업로드해 관련 조항과 근거를 확인하고, Claude/GPT 초안과 교차 검토해 내부 기준과 맞지 않는 표현을 조정
정책 문구 검토 GPT/Claude로 문구 초안을 정리한 뒤, NotebookLM으로 약관·정책 원문 근거를 재확인해 확정 표현과 책임 주체 표현을 조정
업무 보고 자동화 일일 운영 현황 집계와 보고 초안 작성에 AI를 보조 도구로 활용해 반복 작업 시간 단축
